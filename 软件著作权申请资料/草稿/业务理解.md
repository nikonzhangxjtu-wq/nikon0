# 业务理解

- 软件名称：多模态智能客服Agent系统
- 产品定位：面向电商及消费电子领域的多模态智能客服Agent系统，通过融合产品手册检索增强生成（RAG）、多模态图像理解、多轮对话记忆与工单收集能力，为消费者提供产品使用咨询、故障排查、售后受理等一站式智能客服体验。
- 面向领域 / 行业：电商/消费电子/智能客服
- 核心价值：以检索增强生成技术为核心，将海量产品说明手册索引化为可检索知识库，结合多模态视觉理解处理用户上传的图片，通过意图路由自动判断问题类型并选择最佳回答策略，同时支持多轮对话记忆和工单收集流程，提升客服效率和回答准确率。
- 证据文件：`业务理解证据.md`

## 目标用户

- 电商平台消费者（查询产品说明书、故障排查）
- 品牌客服人员（辅助回答产品问题）
- 系统运维人员（部署配置、索引管理、评测优化）

## 主要业务功能

- 智能意图路由
- 产品手册检索问答
- 多模态图像理解
- 工单收集与售后受理
- 在线评价查询
- 多轮对话记忆
- 问题改写优化检索
- 批量问题处理
- 离线评测优化

## 功能说明

- 智能意图路由：系统通过关键词加权打分和LLM仲裁，自动判断用户问题属于产品说明书查询、客服政策咨询、售后工单受理还是未知领域，据此决定是否需要检索产品手册知识库以及采用何种回答策略。
- 产品手册检索问答：系统将20余类产品说明书（家电、数码、运动器械等）解析为文本块，通过BM25稀疏向量和Dense稠密向量混合检索，结合精排模型对召回结果重排序，从最相关的产品手册段落中提取信息生成准确回答。
- 多模态图像理解：系统支持用户上传最多3张Base64编码的图片，通过视觉语言模型提取图片中的文字OCR、关键实体和产品类型，将视觉信息融入检索查询和答案生成，帮助定位产品型号和故障现象。
- 工单收集与售后受理：当用户表达报修、退货、退款等售后意图时，系统自动进入工单收集流程，通过多轮对话逐步收集产品型号、购买渠道、故障现象、联系方式等必要信息，完成工单草稿。
- 在线评价查询：系统支持查询产品的在线评价和用户口碑信息，帮助消费者了解产品优缺点和真实使用反馈，辅助购买决策。
- 多轮对话记忆：系统记录用户会话历史，提取关键事实和用户偏好形成长期记忆，在后续对话中自动加载相关记忆上下文，支持跨会话的个性化服务。
- 问题改写优化检索：当存在对话历史或记忆上下文时，系统自动将用户当前问题改写为更完整、更适合检索的查询语句，提升检索召回准确率。
- 批量问题处理：系统支持从CSV文件读取大量用户问题，批量执行对话流水线生成答案，输出符合竞赛提交格式的结果文件。
- 离线评测优化：系统内置离线评测框架，支持在评测集上运行流水线并记录路由决策、检索命中率、答案质量分数等指标，便于迭代优化系统性能。

## 典型操作流程

1. 用户通过HTTP POST /chat接口提交问题（可附带图片和会话ID）
2. 系统验证Bearer Token并确保会话ID存在
3. 如有历史对话和长期记忆，加载相关上下文并改写问题
4. 视觉模块处理用户上传的图片，提取OCR文字和关键实体
5. 路由模块分析问题意图（说明书查询/客服咨询/工单受理/未知）
6. 不需要检索时，直接使用客服话术模板生成回答
7. 需要检索时，先确定目标产品手册，再执行混合检索和多路召回
8. 对召回结果执行精排和过滤，保留最相关的文本块
9. 组装Prompt（用户问题+检索上下文+视觉摘要+对话历史+记忆）
10. LLM根据Prompt生成最终答案并返回

## 操作手册页面/流程模块

1. 系统部署与启动：在项目根目录下，通过终端命令行执行启动指令。
   - 使用场景：运维人员在新服务器或开发环境中首次部署系统，或系统升级后重新启动服务时使用此流程。
   - 操作要点：确认已安装Python 3.10和所需依赖包；编辑.env文件，配置API端口、Bearer Token、Ollama服务地址、生成模型名称、嵌入模型名称、Milvus数据库连接等参数；在终端中执行 uvicorn app.main:app --reload --host 0.0.0.0 --port 8000 启动API服务；观察终端日志输出，确认服务已成功启动并监听指定端口
2. API对话接口调用：通过HTTP客户端（如curl、Postman或前端应用）发送POST请求至 /chat 接口。
   - 使用场景：消费者在电商平台或客服对话界面提交产品使用问题时，由前端或中间层调用此接口获取智能回答。
   - 操作要点：在HTTP请求头中设置 Authorization: Bearer <token>，token需与.env中配置的API_BEARER_TOKEN一致；构建请求体JSON，填写question字段为用户实际问题，可选附带images图片列表和session_id会话标识；发送POST请求至 /chat 接口；系统处理后返回JSON响应，从data.answer字段获取回答文本，data.session_id用于后续同一会话的继续对话
3. 产品手册索引构建：在项目根目录下通过终端执行索引构建命令。
   - 使用场景：运维人员在首次部署系统或新增产品手册时，运行索引构建脚本将手册内容导入向量数据库。
   - 操作要点：将要索引的产品说明书.txt文件放入手册目录（默认 ./手册/）；确认Milvus数据库服务正在运行且可连接；在终端中执行 python scripts/build_index.py 启动索引构建；观察终端日志，确认手册文件解析、文本切块、向量嵌入和Milvus写入各步骤均成功完成
4. 批量问题处理：在项目根目录下通过终端执行批量处理命令。
   - 使用场景：运维或评测人员在收集到一批用户问题后，使用批量处理功能统一生成答案，用于竞赛提交或效果评估。
   - 操作要点：准备符合格式的CSV输入文件，至少包含id列和question列；确认API服务依赖（Ollama、Milvus）均已正常运行；在终端中执行 python scripts/run_batch_submission.py 启动批量处理；观察处理进度日志，关注每个问题的处理状态和耗时
5. 离线评测：在项目根目录下通过终端执行评测命令。
   - 使用场景：系统开发或优化后，运维或评测人员运行离线评测对比不同版本的效果，指导后续优化方向。
   - 操作要点：准备评测集JSONL文件，确保包含id、question等必要字段；在终端中执行 python -m eval.run_eval --version <版本标识> 启动评测；可添加 --no-judge 参数跳过LLM打分仅统计路由和检索指标；评测完成后查看 results/ 目录下的detail和summary文件
6. MCP网关管理：在mcp-gateway目录下通过Docker Compose启动网关服务，或通过HTTP接口管理下游MCP服务。
   - 使用场景：运维人员在需要集成外部工具能力（如地图查询、网页内容抓取、代码沙盒执行）时，部署和管理MCP网关。
   - 操作要点：在mcp-gateway目录下执行 docker compose -f docker-compose.java-mcp-gateway.yml up -d --build 构建并启动网关；执行 docker compose ps 检查Gateway和Nginx容器是否均正常运行；访问 http://localhost:18080/health 验证网关健康状态；通过Gateway的MCP接口查询可用服务列表、查看服务工具、测试工具调用

## 申请表建议口径

- 开发目的：构建面向电商消费电子领域的智能客服系统，通过产品手册知识检索和多模态理解技术，为消费者提供准确、及时的产品使用咨询和售后服务。
- 软件的主要功能：本系统是一款面向电商及消费电子领域的多模态智能客服Agent软件。系统核心功能包括：（1）智能意图路由：通过关键词打分和LLM仲裁自动判断用户问题类型，决定是否检索产品手册知识库。（2）产品手册检索问答：将20余类消费电子产品说明书索引至向量数据库，支持BM25稀疏检索、Dense稠密检索和精排模型重排序，从最相关的产品手册段落中提取信息生成准确回答。（3）多模态图像理解：支持用户上传产品图片、故障照片等，通过视觉语言模型提取OCR文字和关键实体，融入检索和生成流程。（4）工单收集：当用户表达报修、退货等售后意图时，自动进入多轮信息收集流程，逐步获取工单所需字段。（5）在线评价查询：支持查询产品评价和口碑信息。（6）多轮对话记忆：记录用户会话历史和长期偏好，在后续对话中自动加载相关上下文。（7）问题改写：利用对话历史自动改写用户问题以提升检索效果。（8）批量处理与离线评测：支持CSV批量问题处理和离线指标评测，便于系统迭代优化。
- 技术特点：采用检索增强生成架构，结合关键词路由加LLM仲裁的混合意图识别，支持BM25加Dense多路混合检索与Cross-Encoder精排，集成视觉语言模型实现多模态理解，具备分级记忆系统，通过MCP协议网关集成外部工具能力。
- 软件的技术特点选项：应用软件
- 软件分类：应用软件
